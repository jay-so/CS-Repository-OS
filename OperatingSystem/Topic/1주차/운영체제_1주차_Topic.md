# 📝 운영체제 1주차 질문

---

## 1. 운영체제는 무엇이고 어떤 역할을 수행하는지 설명해주세요.

```text
운영체제란 컴퓨터 하드웨어 바로 위에 설치되어, 사용자 및 다른 모든 소프트웨어와 하드웨어를 연결하기 위한 계층입니다.
사용자는 운영체제를 통해 소프트웨어를 사용할 수 있으며, 해당 소프트웨어를 통해 하드웨어의 작업 결과를 얻을수 있습니다.
이처럼 운영체제는 컴퓨터의 작업을 효율성과 형평성에 맞게 분배하는 역할을 합니다.
```

## 1-1. (꼬리 질문) 운영체제는 어떻게 분류할 수 있을 까요?

```text
운영체제는 동시 작업 가능 여부에 따라 단일 작업용 운영체제, 다중 작업용 운영체제로 분류할 수 있으며
다중 사용자 지원 여부로 단일 사용자용 운영체제, 다중 사용자 지원 운영체제로 분류할 수 있습니다. 
또한 작업 처리 방법에 따른 분류로 일괄처리 방법과 시분할 처리방법으로 구분할 수 있으며
정해진 시간안에 반드시 수행되어야 하는 실시간 처리가 필요한 운영체제로 분류할 수 있습니다.
```

## 1-2. (꼬리 질문) 시분할 운영체제가 빠를 까요? 아님 실시간 운영체제가 빠를까요?

```text
실시간 운영체제는 주어진 시간안에 무조건 해당 작업이 모두 처리가 되어야 함으로 시분할 운영체제보다
실시간 운영체제가 빠릅니다.
```

## 1-3. (꼬리 질문) 멀티 테스킹, 멀티 프로그래밍, 타임 쉐어링, 멀티 프로세스라는 용어의 공통점은 무엇입니까?

```text
멀티 테스킹, 멀티 프로그래밍, 타임 쉐어링, 멀티 프로세스라는 용어의 공통점은
모두 동일하게 컴퓨터 안에서 여러 프로그램이 작동된다는 점입니다.
```

## 1-4. (꼬리 질문) 운영체제는 공개 소프트웨어와 비공개 소프트웨어가 있는데, 왜 사람들은 무료로 운영체제의 소스 코드를 공개했을까요?

```text
소프트웨어는 개발하는데, 시간 및 인건비가 드나 개발한 후 판매 시 재료비 등의 부가적인 기능이 없기 때문에
사용자들로부터 시장의 점유율을 확보하기 위해 무료로 소스 코드를 공개합니다.
```

## 2. 시분할 시스템에 대해서 설명해주세요.

```text
시분할 시스템이란 다중 작업용 운영체제에서 여러 프로그램 작업을 수행할 때, 
한 개의 CPU의 작업 시간을 여러 프로그램들이 조금씩 나누어서 사용하는 시스템을 의미합니다.
```

## 3. 다중 프로그래밍 시스템(multi-programming system)에 대해서 설명해주세요.

```text
CPU와 달리 메모리의 경우, 여러 프로그램을 조금씩 메모리에 올려두고 처리할 수 있습니다.
이처럼 메모리에 여러 프로그램을 조금씩 올려두고, 처리하는 시스템을 다중 프로그래밍 시스템이라고 합니다.
```

## 4. 대화형 시스템(interactive system)에 대해서 설명해주세요.

```text
다중 작업용 운영체제는 여러 프로그램을 실행시키지만, 사용자 관점에서는 각 프로그램에 대한 입력에 대한 결과를 
입력한 후, 바로 보여주기 때문에 이를 마치 대화하는 듯한 대화형 프로그램이라고도 합니다. 
```

## 4-1.(꼬리질문) 다중 작업, 시분할, 다중 프로그램, 대화형 프로그램의 공통점은 무엇인가요?

```text
모두 프로그램이 하나의 컴퓨터에 실행된다는 공통점이 있습니다.
```

## 5. 다중 처리기 시스템(multi-processor system)에 대해서 설명해주세요.

```text
다중 처리기 시스템이란 하나의 컴퓨터에 여러 개의 CPU가 설치된 시스템을 의미합니다.
해당 시스템에서의 각 CPU는 돌립적으로 작동하여 여러 작업을 동시에 처리할 수 있습니다.
그러나 운영체제는 각 CPU를 효율적으로 관리하고 동기화하는 복잡한 메커니즘이 필요합니다.
```

## 6. 시스템 콜에 대해 설명해주세요.

```text
시스템 콜이란, 사용자 프로그램이 운영체제 내부에 정의된 코드를 실행하고 싶을 때
운영체제에게 작업 처리 요청을 넘기는 과정을 부탁하기 위한 요청 과정입니다.
```

```text
이를 메모리 상에 올라온 운영체제에게 부탁하여 처리합니다.
```

## 6-2. (꼬리 질문) 운영체제에게 CPU의 주도권이 넘어가는 3가지 경우에 대해서 설명해주세요.

```text
그 다음으로 사용자 프로그램이 운영체제에게 작업을 부탁을 할 때, 시스템을 호출하는 경우와 함께
0으로 나누는 연산, 권한이 없는 기계어를 순간과 같은 예외가 발생되는 경우에 운영체제에게 CPU의 주도권이 넘어갑니다.
```

## 6-3. (꼬리 질문) 사용자 프로그램에게 CPU 주도권이 있다가, 운영체제에게 넘어가는 2가지 경우에 대해서 설명해주세요.

```text
먼저, 사용자 프로그램이 CPU를 계속 사용할 경우, 타이머에 의해 인터럽트가 발생되어 프로그램이 주도권이 운영체제에게 넘어가는 경우가 있습니다.
그 다음, I/O 작업에 있어, 다음 작업에게 CPU 주도권을 넘겨줘야 할때 사용자 프로그램에서 운영체제로 주도권이 넘어갑니다.
```

## 10. 폴링에 대해 설명해주세요.

```text
폴링은 CPU가 주기적으로 I/O 장치의 상태를 검사하며
데이터 처리가 필요한 시점을 판단하는 방식을 말합니다.
이는 CPU가 직접 I/O 장치에 접근하여 데이터를 가져오거나 내보내는 방식입니다.
CPU는 I/O 장치의 상태를 지속적으로 모니터링하며, 특정 조건이 충족될 때 데이터를 처리합니다.
하지만 이 방식은 CPU의 지속적인 모니터링 작업으로 인해 작업 효율이 떨어질 수 있습니다.
따라서, 현재의 다양한 프로세스를 처리하는 데는 폴링 방식이 적합하지 않을 수 있습니다.
```

## 11. 인터럽트에 대해 설명해주세요.

```text
인터럽트는 I/O 장치에서 특정 작업이 완료되었을 때나 특정 상황이 발생했을 때 발생하는 신호입니다. 
해당 신호는 디바이스 컨트롤러에 의해 생성되며, 인터럽트 라인을 통해 CPU에 전달됩니다. 
이때, CPU가 이 인터럽트 신호를 받으면
현재 수행 중인 작업의 위치를 Program Counter (PC)에 저장하고, 현재 작업을 잠시 중단합니다. 
그 후 CPU는 인터럽트로 전달된 작업, 즉 I/O 작업의 결과를 
로컬 버퍼에서 메인 메모리로 옮기는 등의 작업을 우선적으로 수행합니다. 
해당 작업이 완료되면, CPU는 PC에 저장된 위치로 돌아가 원래의 작업을 재개합니다. 
```

## 11-1. (꼬리 질문)인터럽트 처리 루틴이란 무엇인지 설명해주세요.

```text
인터럽트 처리 루틴이란, 운영체제 커널에 인터럽트가 들어왔을 때 
특정 인터럽트에 대응하여 실행되어야 하는 코드를 의미합니다.
```

## 11-2. (꼬리 질문) 하드웨어 인터럽트와 소프트웨 인터럽트의 공통점에 대해서 설명해주세요.

```text
하드웨어 인터럽트와 소프트웨어 인터럽트의 공통점은 인터럽트 라인에 신호를 보내 
인터럽트가 발생되었다는 알려주는 공통점이 있습니다. 
```

## 11-3. (꼬리 질문) 인터럽트 백터와 인터럽트 핸들러에 대해서 설명해주세요.

```text
또한, 인터럽트 백터에 따라 어떤 일을 해야 하는지 정의되어 있는 코드를 인터럽트 핸들러라고 부릅니다.
```

## 12. DMA에 대해 설명해주세요.

```text
DMA(Direct Memory Access)는 I/O 장치가 CPU의 개입 없이 
직접 메인 메모리에 데이터를 읽고 쓸 수 있게 하는 기술을 말합니다.
디바이스 컨트롤러는 CPU의 중재 없이 로컬 버퍼의 내용을 Block 단위로 메인 메모리에 직접 전송합니다.
해당 과정에서는 인터럽트를 발생시키지 않아, CPU는 다른 작업을 계속 수행할 수 있습니다.
전송 작업이 모두 끝나면, DMA 컨트롤러는 CPU에 인터럽트를 발생시켜 전송이 완료되었음을 알립니다.
이를 통해 CPU는 한 번에 전송된 데이터 블록을 처리하게 됩니다. 
이렇게 함으로써, DMA는 CPU의 부하를 줄이고 전체 시스템의 효율성을 향상시킵니다. 
```

## 13. 동기식 I/O에 대해 설명해주세요.

```text
동기식 입출력은 프로그램이 I/O 작업을 요청하고, 해당 작업이 완전히 완료된 후에야 프로그램이 다시 수행되는 방식을 말합니다.
이는 두 가지 방식으로 구현될 수 있습니다.
첫 번째 방식은 프로그램이 I/O 요청을 한 후, 해당 작업이 완료될 때까지 CPU를 점유하며 대기하는 방식입니다.
해당 방식의 단점은 I/O 작업이 완료될 때까지 CPU를 사용하지 못하며, 한 번에 하나의 I/O만 처리할 수 있다는 점입니다.
두 번째 방식은 I/O 작업이 완료될 때까지 해당 프로그램이 CPU의 제어권을 다른 프로그램에게 양보하고 대기하는 방식입니다.
해당 방식의 장점은 다른 I/O 장치를 실행시킬 수 있다는 점입니다.
```

## 13-1. (꼬리 질문) 그렇다면 두 번째 동기식 I/O 작업에서의 단점에 대해서 설명해주세요.
```text
I/O 작업이 완료될 때까지 프로그램은 작업 완료를 기다리는 블로킹 상태에 머물게 되는 단점이 있으며
다른 프로그램에게 CPU를 양보할 때, 운영체제는 컨텍스트 스위칭을 수행함으로 오버헤드가 발생될 수 있습니다.
```


## 14. 비동기식 I/O에 대해 설명해주세요.

```text
비동기식 입출력은 프로그램이 I/O 작업을 요청한 후,
해당 작업의 완료 여부와 상관없이 프로그램이 다른 작업을 계속 진행할 수 있게 하는 방식을 말합니다. 
이는 I/O 작업이 완료되기 전에도 다른 프로그램에게 CPU의 주도권을 넘겨주어 처리할 수 있습니다.
해당 방식은 논 블로킹 상태를 유지하면서 CPU 사용을 효율적으로 할 수 있다는 장점이 있지만,
동기식 입출력에 비해 프로그래밍이 복잡하다는 단점이 있습니다.
```

## 16-1. (꼬리 질문) 컴퓨터 내부에서 CPU가 연산중일 때, 외부에서 디바이스 컨트롤러가 I/O 장치의 연산을 동시에 수행하는 것이 가능한가요?

```text
네, 가능합니다. 
CPU와 디바이스 컨트롤러는 독립적으로 작동하기 때문에, 
CPU가 연산을 수행하는 동안 디바이스 컨트롤러는 동시에 I/O 장치의 연산을 수행할 수 있습니다. 
```

## 16-2. (꼬리 질문) 디바이스 드라이버와 디바이스 컨트롤러에 대해서 설명해주세요.

```text
디바이스 드라이버란, 운영체제 안에 있는 코드이며, I/O 컨트롤러에게 해당 작업을 넘기는 코드를 의미합니다.
소프트웨어이며, CPU 내부에서 작업이 일어납니다.
반면 디바이스 컨트롤러는 프로그램으로부터 요청이 들어온 I/O 작업을 처리하는 컨트롤러를 의미합니다.
하드웨어이며, CPU 외부에서 작업이 일어납니다.
```
